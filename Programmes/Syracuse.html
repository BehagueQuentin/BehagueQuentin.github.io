Suite de Syracuse

Vous trouverez sur cette page un petit programme que j'ai implémenté en terminal dans le cadre de mon grand oral.
Il vous permettra de visualiser la suite de Syracuse d'une graine choisie.
À l'avenir, des programmes visant à vérifier la conjecture de Syracuse sur un grand nombre d'entiers seront ajoutés sur cette page.
Ils seront plus complexes que ceux déjà présents, et les raisonnements permettant de les établir seront détaillés.

  • Un premier code élémentaire :

    • En utilisant simplement la définition des suites de Syracuse, on obtient un premier programme qui part de toute graine comprise entre 1 et l'entier choisi et qui applique la relation de récurrence jusqu'à obtenir un 1. 

    
def verification_syracuse_0(n):
    for k in range(1, n+1):
        while k != 1:
            if k % 2 == 1:
                k = 3*k + 1
            else:
                k = k // 2
    return True
  • Un simple constat de parité :

    • Dans le cas où l'entier \(k_0\) considéré est impaire, le nombre obtenu par l'opération \(k_1 = 3 \times k_0 + 1\) est nécessairement pair.
    On peut alors s'épargner une vérification de parité en modifiant légérement le code précédent : 

    
def verification_syracuse_1(n):
    for k in range(1, n+1):
        while k != 1:
            if k % 2 == 1:
                k = (3*k + 1)//2
            else:
                k = k // 2
    return True
  • Diminuer le nombre de calculs en se servant d'une équivalence :

    • Démonstration de l'équivalence :

      Conjecture de Syracuse implique temps de vol en altitude fini :

      Ce sens est trivial : considèrons la suite de Syracuse de graine \(k > 1\), si la conjecture de Syracuse est vraie, cela signifie qu'il existe un entier \(n\) pour lequel \(S_n = 1\).
      En particulier, cela implique que le temps de vol en altitude est fini, car il est strictemment inférieur à \(n\).

      Temps de vol en altitude fini implique Conjecture de Syracuse :

      Pour la réciproque, on procède par récurrence forte.
      On prend pour hypothèse de récurrence \(H_n\) : "La suite de Syracuse de graine \(k\) admet un temps de vol fini".
      Pour l'initialisation, si \(k=1\) la conjecture est vraie car la suite atteint 1 immédiatement.
      Pour l'hérédité, soit \(k>1\), par hypothése de temps de vol en altitude fini, soit \(n_0 \in \mathbb N\) le premier indice tel que \(S_{n_0} < k\).
      Or, par hypothèse de récurrence, la suite de graine \(S_{n_0}\) admet un temps de vol fini, soit \(n_1\) le premier indice tel que \(S_{n_1} = 1\). Alors, le temps de vol de la suite de Syracuse de graine \(k\) vaut \(n_0 + n_1\), il est donc fini. D'où le résultat par principe de récurrence. 

    
def verification_syracuse_2(n):
    for k in range(2, n+1):
        k0 = k
        while k >= k0:
            if k % 2 == 1:
                k = (3*k + 1)//2
            else:
                k = k // 2
    return True
  • Temps de vol en altitude fini selon la congruence :

    • En étudiant le vol des nombres selon leur congruence par des entiers pairs, il est possible d'établir que certain d'entre eux vont nécessairement avoir un temps de vol en altitude fini.
    En effet, on remarque que les entiers pairs ont un temps de vol en altitude égal à 1, il n'est donc pas nécessaire de les traiter.
    De même si on considère les entiers congrus à \(1\) modulo \(4\) : \[4k + 1 \stackrel{impair}{\longrightarrow} 12k + 4 \stackrel{pair}{\longrightarrow} 6k + 2 \stackrel{pair}{\longrightarrow} 3k + 1\] Or, pour tout \(k \geq 1\), \(4k + 1 > 3k + 1\).Leur temps de vol en altitude vaut 3, il n'est pas nécessaire de les tester.
    Si on tente de faire de même avec les entiers congrus à \(3\) modulo \(4\), on tombe dans une impasse : \[4k + 3 \stackrel{impair}{\longrightarrow} 12k + 10 \stackrel{pair}{\longrightarrow} 6k + 5 \stackrel{impair}{\longrightarrow} 18k + 16 \stackrel{pair}{\longrightarrow} 9k + 8\] On ne connait pas la parité de \(9k + 8\), et \(9k + 8 > 4k + 3\) pour tout \(k \geq 1\). On ne peut rien en déduire.
    On est alors tenté de regarder ce qui ce produit en fonction de la parité de \(k\).
    Autrement dit, on considère les entiers congrus à \(3\) ou \(7\) modulo \(8\).
    On se retrouve une fois de plus dans une impasse. on passe à la puissance de deux supérieure
    \[16k + 3 \stackrel{impair}{\longrightarrow} 48k + 10 \stackrel{pair}{\longrightarrow} 24k + 5 \stackrel{impair}{\longrightarrow} 72k + 16 \stackrel{pair}{\longrightarrow} 36k + 8 \stackrel{pair}{\longrightarrow} 18k + 4 \stackrel{pair}{\longrightarrow} 9k + 2\] 

    
def verification_syracuse_3(n):
    for k in range(2, n+1):
        cg = k%16
        if cg == 7 or cg == 11 or cg == 15:
            k0 = k
            while k >= k0:
                if k % 2 == 1:
                    k = (3*k + 1)//2
                else:
                    k = k // 2
    return True
